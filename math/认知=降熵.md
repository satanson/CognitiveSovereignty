
## 从最小完备集到高维正则化的思维转向

大学的数学教育成功地将“最小完备集”作为思想钢印植入我们的大脑，后续工作中又不断地强化。进入这种歧途之后，我们会认为最小完备集是最好的，甚至害怕升维扩域带来的更高复杂性。然而，除了最小完备集之外，高维正则化也是一种有效的降熵方式。

在探索证明 $k$ 阶 Cauchy-Binet 公式的过程中，前期的思维依然受困于最小完备集，即便处理 2 阶公式也面临符号和公式的爆炸，大脑带宽根本无法负载。经过痛苦的尝试，最终使用高维正则化的方式才成功完成了证明。该公式作为外代数的核心公式之一，其实很容易找到现成的证明过程，但 follow 这个现成的证明并不是动机；通过反复拆解，使思维从最小完备集逐渐过渡到高维正则化，并且验证高维正则化是确实可行的方法，这才是真正的目的。历经的痛苦，恰恰强化了对拆解过程的记忆与深入理解。

这种思维的对立，在程序员处理多分支逻辑时同样存在：

1. **最小完备集思维**：采用多重 `if-else` 分支处理。每个分支里都包含当前条件下的特例化处理逻辑，这种代码可维护性极差，难以理解。
2. **高维正则化思维**：将多重 `if-else` 改写，通过回调的方式注入特例处理逻辑，并将这些特例逻辑统一注册到表（Table）中，采用表驱动的方式工作。注册表通常是稀疏的，只有部分项代表实际的计算逻辑，但我们通常会主动引入 `no-op` 的平凡函数（Trivial Function）去填充空项，让表的整个结构保持“正则”。这种思维的转变，给表的整体和元素都赋予了全新的概念和语义，反而更容易理解，也更利于后续演化。

## 最小完备集的局限性
如果把学习看成是建一座房子：立基、建造、封顶，整个过程的全面掌握非常关键；然而，我们目前其实只看到了成品的房子。
我在思考一个问题：在我的认知里，归纳和演绎，到底哪一个更重要呢？
但其实，我们的思考过程其实是“探索 + 归纳 + 少量演绎”；最后的封顶才是纯粹的演绎——以此把结论做实，使得立基稳固。

但是这样带来了一个问题，就是：

1. **立基时的局限**：在立基时，我们的思路一直是最小完备集，也就是公理化。仿佛只有当最后的封顶演绎能够用这个最小完备集完全表达出来时，这个真理才能被成功“封印”。这里说的最小完备集，可以概括为两个基本方法 **hoisting（提升）** 和 **lowering（下沉）**：`高层概念 = 别名(编排(底层概念))`；hoisting 就是把底层概念 rollup，lowering 就是把高层概念 drill down。
2. **教学的倒置**：在教学时，学校呈现给我们的往往是最终的成品，而不是建造的过程。这个成品是被公理化封印过的完美结果，导致我们根本不知道这个思考过程是如何达成的。
3. **思维的误导**：我们的思维，恰恰被这种对“最小完备集”的执念引入了歧途。

我们思维之所以会陷入这种模式，有下面几个深层原因：

1. 我接触的正统教育是完全按照公理化思路编写的；
2. 我们是工程师，工程上的冗余通常只作容错考虑，在思考工程架构时，我们也是按照 Unix 的 KISS（至简）思维来处理的；
3. 我们是搞计算机的，编译优化、SQL 处理、系统开发，其实头脑里根深蒂固的认知模式是图灵机；
4. 现代数据处理借鉴的Lisp设计思想;
4. 我们的思维是高度基于构造性的。

## 高维正则化
学习的本质是降熵。最小完备集这种降熵思路，对于学习数学不是唯一有效的方式，还有另外一种同样有效的方式，这种方式就是**高维正则化**。

建造的过程，其实是给数学法则寻找**正则代数空间**。
高维正则化，才是学习数学的真正门径。比如验算 Cauchy-Binet 公式的方式，其核心思路是：通过正则化、增加冗余得到了一个新的代数结构，新的代数结构以某种方式可以分解，然后这个代数结构上的某种性质，对于代数结构的分解，展现出了完美的**分解重组性**：

1. **正则化**：$A = \text{残缺 } A \cup \{\text{平凡项}\}$；
2. **系统内分解**：分解正则 $A$。$A$ 在自己的代数系统内，针对运算 $\times$ 满足封闭性：$A = A_0 \times A_1 \times A_2 \times A_3$；
3. **空间映射**：存在一个映射 $T$，可以将 $A$ 映射到另外一个空间 $B$，$B$ 中存在运算 $+$。这个映射满足同态性质：$T(A) = T(A_0) + T(A_1) + T(A_2) + T(A_3)$；
4. **降维打击**：空间 $B$ 的性质良好，计算极其简单。

将问题转换为高维，引入平凡项，使问题正则化，然后观察高维代数系统的规则是什么，进而运用这个规则去研究和解决问题。这就好比玩一个魔方：如果把它随机乱拧后剪开，将六个面平摊在平面上，我们将什么规律都看不出来；但是，如果把它重新拼回三维六面体，然后去证明这个六面体可以通过特定的拧法最终恢复成六面同色，一切就变得显而易见了。
